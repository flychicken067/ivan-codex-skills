# Gotchas

## Trigger Boundaries

- Use this skill only for visual work.
- Do not auto-apply it to backend, API, database, or CI tasks.

## Common Failure Patterns

- Centering everything by default when asymmetry would read better.
- Replacing meaningful product visuals with fake screenshots or decorative clutter.
- Using muddy border plus shadow stacks instead of a crisp ring treatment.
- Giving aesthetic advice without concrete code-level changes.

## Verification Rules

- If images, logos, or testimonial tracks exist, verify that every expected asset renders.
- If the request mentions missing tracks such as animal, abstract, or business, explicitly confirm all of them.
