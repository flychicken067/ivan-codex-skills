# Case Library

Use these as baseline scenarios for fast, repeatable evaluation.

## Case A: Marketing Hero Cleanup

- Input shape: SaaS homepage with generic hero and fake UI mock.
- Common failure: centered-everything, purple drift, oversized CTA chrome.
- Must pass:
  - stronger hierarchy
  - neutral/brand-aligned palette
  - real screenshot treatment and crisp ring/shadow stack

## Case B: Dashboard Landing Polish

- Input shape: feature-heavy page with screenshot cards and stats.
- Common failure: muddy borders, noisy spacing rhythm, mixed gray families.
- Must pass:
  - coherent spacing grid
  - typography tuning (weight/tracking/measure)
  - polished cards and readable stat hierarchy

## Case C: Testimonial Media Recovery

- Input shape: testimonial section with background portraits/assets.
- Common failure: two tracks missing due to path/case/format or layering.
- Must pass:
  - all tracks visible: animal, abstract, business
  - no hidden/clipped/zero-height media
  - verification checklist included in handoff
